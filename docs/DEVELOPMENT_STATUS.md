## 現在の注意点

- データはブラウザ単位で保存され、別端末とは同期されない
- サイトデータを削除するとメモとToDoが失われる可能性がある
- 自動テストは未導入で、主な確認方法は型チェック、ビルド、ブラウザ操作
- `Sidebar.tsx` と `TodoSidebar.tsx` は約1,000行あり、今後さらに複雑になる場合は部分的な分割を検討する
- `.vscode/` は現在Git管理外。共有設定として必要になるまでコミットしない
- Tiptapのリンク拡張について、開発時に重複警告が出る場合があるため、エディタ周辺を変更するときに確認する

## 次の開発候補

優先順位は固定していません。次の作業開始時に、目的と学習内容を見て1つ選びます。

1. 主要なデータ操作のテスト追加
2. サイドバー内の重複UIを小さな共通部品へ分割
3. 削除後に元へ戻せるUndo操作
4. ToDoの期限・優先度
5. localStorageデータのバックアップと復元

バックアップ、画像添付、クラウド同期は検討済みですが、シンプルさを優先して現時点では保留しています。

## 作業再開時の確認

```bash
git status -sb
npm install
npm run dev
```

必要に応じて次も実行します。

```bash
npm run typecheck
npm run build
```

## 作業終了時の確認

1. 変更した機能を画面で確認する
2. `npm run typecheck`を実行する
3. リリース前や共有処理の変更では`npm run build`も実行する
4. この資料の「現在の状態」「注意点」「次の開発候補」に変化があれば更新する
5. 関係するファイルだけをコミットする
